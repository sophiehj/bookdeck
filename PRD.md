**작성일:** 2026-04-18  
**작성자:** sophie.hjpark  
**스택:** React 19 · TypeScript · Vite · Firebase · Claude API

## 1. 제품 개요

### 1.1 한 줄 정의

결정 장애 독자를 위한 **한 번에 한 권씩** 설득형 도서 추천 서비스 — 리스트 탐색 없이 "지금 읽을 한 권"을 확정한다.

### 1.2 기존 서비스와의 차별화

| | 온라인 서점 / 기존 독서 앱 | **Booklip** |
|---|---|---|
| UX 패러다임 | 내가 찾는다 (검색 → 리스트) | 앱이 설득한다 (카드 한 장씩 제안) |
| 정보 구조 | 줄거리 + 별점 + 리뷰 수 | 핵심 요약 + 강렬한 구절 + **나에게 맞는 이유** |
| 목표 | 책 발견 | **"지금 읽을 한 권" 결정** |
| 사용자 역할 | 판단자 (검색·비교) | 반응자 (읽을래요 / 패스만) |
| 학습 | 없음 | 피드백 → 다음 카드 즉시 개선 |

### 1.3 핵심 가치 제안

> "고르는 고통은 끝, 읽는 즐거움만 남긴다."  
> 침대에서 책 고르다 1시간 날리는 경험을 없앤다.

### 1.4 타깃 사용자

- **주 타깃:** 25–40세, 읽고 싶은 마음은 있지만 결정 장애를 겪는 직장인
- **부 타깃:** 독서 모임 운영자 (AI 모더레이터 활용), 도서 큐레이터

## 2. 핵심 사용자 여정

```
앱 열기
    ↓
추천 카드 한 장 등장
  ┌─ 카드 내용 ──────────────────────────────────┐
  │  • 강렬한 한 문장 훅 (hero)                  │
  │  • 핵심 요약 3줄 (줄거리 / 메시지 / 추천 독자)│
  │  • "당신에게 맞는 이유" (맞춤형 1줄)          │
  └──────────────────────────────────────────────┘
    ↓
[읽을래요 ✓] 또는 [패스 →]
    ↓
피드백 저장 → 다음 카드가 더 정교해짐
    ↓
"읽을래요" 선택 시 → 책 상세 페이지
    ↓
서평 작성 + 감정 반응 (공감 / 반박 / 인상적)
    ↓
독서 트윈 매칭 → 팝업 독서 모임
```

## 4. 기능 요구사항

---

### Phase 1 — MVP (Week 1–2)

#### 4.1 추천 카드 덱 (핵심 기능)

**FR-001 카드 덱 UI**

- 홈화면 = 카드 한 장 전체 화면 (검색 리스트 아님)
- 카드 구성:
  ```
  ① 강렬한 한 문장 훅     — 큰 타이포, 화면 상단 hero
  ② 핵심 요약 3줄         — 줄거리 / 핵심 메시지 / 추천 독자
  ③ 맞춤형 추천 이유       — "~하는 당신에게 맞는 이유" 1줄
  ④ 책 메타               — 표지 썸네일 · 제목 · 저자
  ```
- 하단 액션 버튼 2개: **[패스 →]** **[읽을래요 ✓]**
- 카드 스와이프 제스처 지원 (좌: 패스 / 우: 읽을래요)

**FR-002 카드 큐 관리**

- 앱 진입 시 카드 5장 미리 로드 (UX 지연 최소화)
- 마지막 카드 도달 시 다음 5장 자동 로드
- 이미 패스하거나 읽을래요한 책은 다시 등장하지 않음

**FR-003 읽을래요 / 패스 피드백**

- 피드백을 Firestore에 저장 (`userFeedbacks/{uid}_{isbn}`)
- 저장 데이터: `{ type: 'want' | 'pass', genre, authors, isbn, timestamp }`
- 읽을래요 → 책 상세 페이지로 이동 + 읽고 싶은 책 목록에 추가
- 패스 → 즉시 다음 카드로 전환

---

#### 4.2 AI 카드 콘텐츠 생성

**FR-010 AI 요약 + 훅 생성**

카카오 도서 API로 책 정보를 가져와 Claude Haiku로 생성:

```json
{
  "hook": "독자를 사로잡는 강렬한 한 문장",
  "plot": "스포일러 없이 세계관·갈등 소개 (2–3문장)",
  "message": "저자가 전하는 중심 통찰 (1–2문장)",
  "recommend": "어떤 독자에게 맞는지 (1문장, ~하는 당신에게)",
  "reason": "이 사용자에게 맞는 이유 (1문장, 피드백 이력 반영)"
}
```

**FR-011 맞춤형 추천 이유 (reason)**

- MVP: 장르·저자 피드백 이력 기반 단순 매칭
  - 패스한 장르 → 줄임 / 읽을래요한 장르 → 늘림
- v0.2: 임베딩 유사도 기반 고도화

**FR-012 콘텐츠 캐싱**

- AI 생성 결과는 Firestore `books/{isbn}/aiSummary`에 캐싱
- 동일 ISBN 재요청 시 LLM 호출 없이 캐시 반환
- 캐시 히트: p95 ≤ 300ms / 미스: p95 ≤ 3초

---

#### 4.3 책 상세 페이지

**FR-020 상세 페이지 구성**

- 카드에서 "읽을래요" 선택 후 진입
- 구성: 훅 hero → 책 메타 → AI 요약 카드 → 서평 목록 → 서평 쓰기

**FR-021 서평 작성**

- 마크다운 에디터
- 별점 (0.5 단위, 0–5) + 완독 여부
- 공개 / 친구 공개 / 비공개
- 빈 에디터 플레이스홀더: "오늘 이 책에서 가장 인상 깊었던 순간은?"

**FR-022 감정 반응**

| 반응 | 의미 |
|---|---|
| 공감 🤝 | 나도 같은 감정을 느꼈다 |
| 반박 💬 | 다르게 읽었다 — 토론하고 싶다 |
| 인상적 ✨ | 이 시각이 새로웠다 |

---

#### 4.4 도서 검색 (보조 기능)

**FR-030 검색**

- 메인 플로우가 아닌 보조 기능 (헤더 검색 아이콘으로 접근)
- 카카오 도서 API, 300ms 디바운스 자동완성
- 검색 결과에서 직접 상세 페이지 진입 가능

---

### Phase 2 — 피드백 학습 & 매칭 (Week 3–4)

#### 4.5 피드백 기반 개인화

**FR-040 추천 고도화**

- 읽을래요/패스 누적 데이터 → 선호 장르·저자 프로파일 생성
- 서평 작성 완료 시 선호도 가중치 추가 반영
- "맞춤형 추천 이유" 구체화: `"IT 직군 분들이 자주 선택한 책"` 등

**FR-041 독서 트윈 매칭**

- 같은 책에 "읽을래요"를 누른 사람들 연결
- 서평 임베딩 코사인 유사도로 독서 궁합 점수 (0–100)
- "책 이야기 나눠요" 1:1 연결 요청

---

### Phase 3 — 모임 (Week 5–6)

#### 4.6 팝업 독서 모임

**FR-050 모임 자동 제안**

- 같은 책 "읽을래요" 4명 이상 → 번개 모임 제안
- 4–8명 규모, 수락 기한 48시간

**FR-051 AI 모더레이터**

- 모임 24시간 전 멤버 서평 분석 → 토론 질문 3개 자동 생성
- 질문 유형: 사실 확인 / 해석 / 삶에 적용

## 5. UI/UX 디자인 시스템

### 5.1 디자인 원칙

- **한 번에 하나:** 화면당 결정해야 할 것 1개 (카드 or 상세 or 서평)
- **설득형 레이아웃:** 훅 → 요약 → 이유 순서로 읽기 동기 상승
- **모바일 First:** iOS Safari / Android Chrome 기준
- **파스텔 안정감:** 독서의 차분한 분위기

### 5.2 컬러 팔레트

| 역할 | 색상 | Hex |
|---|---|---|
| Primary | 라벤더 | `#C3B1E1` |
| Secondary | 민트 | `#B5EAD7` |
| Accent | 피치 | `#FFDAC1` |
| Background | 아이보리 | `#FAFAF7` |
| Surface | 화이트 | `#FFFFFF` |
| Text Primary | 다크 그레이 | `#2D2D2D` |
| Text Secondary | 미디엄 그레이 | `#6B7280` |
| Border | 라이트 그레이 | `#E5E7EB` |

### 5.3 화면 구성

```
[홈 — 카드 덱]
  ├─ 훅 문장 (큰 타이포, hero)
  ├─ 책 표지 + 제목 + 저자
  ├─ AI 요약 3줄
  ├─ 맞춤형 추천 이유
  └─ [패스 →] [읽을래요 ✓]

[책 상세]
  ├─ ← 뒤로가기
  ├─ 훅 hero
  ├─ 책 메타 (표지·제목·저자·출판사)
  ├─ AI 요약 카드
  └─ 서평 목록 + [서평 쓰기]

[보조 — 검색]
  ├─ 검색바
  └─ 검색 결과 리스트

[마이페이지]
  ├─ 읽고 싶은 책 목록 (읽을래요 누른 것)
  └─ 내가 쓴 서평
```

### 5.4 카드 전환 인터랙션

- 스와이프 좌 → 패스 (카드 좌측으로 날아감)
- 스와이프 우 → 읽을래요 (카드 우측으로 날아감, 라벤더 배경 플래시)
- 버튼 탭도 동일 동작

## 6. 기술 아키텍처

### 6.1 프론트엔드

| 항목 | 선택 |
|---|---|
| 프레임워크 | React 19 + TypeScript |
| 번들러 | Vite |
| 상태 관리 | Zustand (전역) + React Query (서버 상태) |
| 스타일링 | Tailwind CSS v4 |
| 라우팅 | React Router v7 |
| 카드 스와이프 | CSS transform + touch events (라이브러리 없이) |

### 6.2 백엔드 & 인프라

| 항목 | 선택 |
|---|---|
| BaaS | Firebase (Auth, Firestore) |
| AI / LLM | Claude Haiku — 요약, 훅, 추천 이유 |
| Embedding | text-embedding-3-small (v0.2, 매칭용) |
| 도서 API | 카카오 도서 API (primary) |

### 6.3 Firestore 데이터 모델

```
users/{uid}
  ├─ profile: { nickname, photoURL }
  ├─ preferences: { likedGenres[], passedGenres[], likedAuthors[] }
  └─ wantToRead: [isbn, ...]

books/{isbn}
  ├─ meta: { title, author, publisher, coverUrl, genre }
  └─ aiSummary: { hook, plot, message, recommend, reason, cachedAt }

userFeedbacks/{uid}_{isbn}
  └─ { userId, isbn, type: 'want'|'pass', genre, timestamp }

reviews/{reviewId}
  ├─ userId, isbn, content, rating, isPublic, isFinished
  └─ reactions: { empathy, counter, impressive }
```

### 6.4 AI 카드 생성 파이프라인

```
카드 덱 요청 (5장)
    ↓
사용자 피드백 이력 조회 (Firestore)
    ↓
[각 책마다]
  캐시 확인 (books/{isbn}/aiSummary)
    ├─ hit  → 캐시 반환
    └─ miss → Claude Haiku 호출 (max_tokens 450)
               → Firestore 캐싱
    ↓
reason 필드만 유저 이력 기반 실시간 생성 (max_tokens 80)
    ↓
카드 5장 반환
```

### 6.5 리팩토링 대비 코드 원칙

| 원칙 | 적용 |
|---|---|
| AI 호출 추상화 | `src/services/ai.ts` 단일 진입점 |
| 도서 API 추상화 | `src/services/bookApi.ts` 인터페이스 고정 |
| 카드 상태 분리 | `src/store/cardStore.ts` — 덱·피드백 전역 관리 |
| 타입 중앙 관리 | `src/types/index.ts` |

## 7. 테스트 전략

| 레이어 | 도구 | 대상 |
|---|---|---|
| Unit | Vitest | services, hooks, utils |
| Component | React Testing Library | 카드 UI, 스와이프 인터랙션, 반응 버튼 |
| Integration | Vitest + fetch mock | 카드 큐 로딩, 피드백 저장 흐름 |

### MVP 핵심 테스트

```
✓ 카드 덱: 5장 로드, 중복 없음
✓ 읽을래요: Firestore에 'want' 저장, 상세 페이지 이동
✓ 패스: Firestore에 'pass' 저장, 다음 카드 전환
✓ AI 요약: 캐시 히트 시 LLM 미호출
✓ 감정 반응: 토글 동작, 카운트 반영
✓ 서평: 플레이스홀더, 등록, 별점
```
