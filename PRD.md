## 1. 제품 개요

### 1.1 한 줄 정의

책을 숏츠처럼 — 분야별 신간을 스와이프로 훑고, 2문장 요약으로 빠르게 접하는 책 발견 서비스.

### 1.2 핵심 가치 제안

> "긴 설명 없이, 딱 두 문장으로 이 책이 나에게 맞는지 바로 안다."  
> 유튜브 숏츠처럼 책을 넘기다 보면 읽고 싶은 책이 생긴다.

### 1.3 타깃 사용자

- 책에 관심은 있지만 어떤 책을 골라야 할지 모르는 독자
- 이동 중·짬 시간에 가볍게 책을 탐색하고 싶은 직장인
- 특정 분야의 신간을 빠르게 파악하고 싶은 독자

## 2. 서비스 플로우

```
앱 로드 (로그인 불필요)
    ↓
[홈 화면]
  ┌─────────────────────────────────┐
  │  🔍 통합검색 (항상 상단 고정)   │
  ├─────────────────────────────────┤
  │  분야 선택                       │
  │  [자기계발] [소설] [에세이]      │
  │  [경제/경영] [역사] [과학] …    │
  └─────────────────────────────────┘
    ↓ 분야 터치
[숏츠 뷰 — 해당 분야 신간]
  ┌─────────────────────────────────┐
  │  🏠 (홈으로)   🔍 통합검색     │ ← 항상 표시
  ├─────────────────────────────────┤
  │                                  │
  │       책 표지 이미지             │
  │                                  │
  │  제목 / 저자                     │
  │  ─────────────────────────       │
  │  2문장 AI 요약                   │
  │                                  │
  └─────────────────────────────────┘
    ↓ 위 또는 옆으로 스와이프
  같은 분야의 다음 책으로 전환
    ↓ 🏠 터치
  홈 화면 (분야 선택)으로 복귀
    ↓ 다른 분야 선택
  새 분야의 신간 숏츠 시작

[검색 플로우]
  통합검색창에 입력
    ↓
  검색 결과의 첫 번째 책 → 숏츠 뷰
    ↓ 스와이프
  같은 분야의 다른 책으로 이어짐
```

## 3. 기능 요구사항

### FR-001 홈 화면 — 분야 선택

- 로그인 없이 즉시 접근 가능
- 상단 고정: 통합검색 바
- 분야 목록 (카드 or 버튼):

| 분야 | 분야 | 분야 |
|---|---|---|
| 자기계발 | 소설 | 에세이 |
| 경제/경영 | 역사 | 과학 |
| 인문 | 사회 | 예술 |

- 분야 터치 → 해당 분야 숏츠 뷰로 진입

---

### FR-002 숏츠 뷰 — 책 카드

- 화면 전체를 하나의 책 카드로 표시
- 카드 구성:
  ```
  ① 책 표지 이미지 (상단, 전체 너비)
  ② 제목 + 저자
  ③ AI 2문장 요약 (핵심 내용 1문장 + 어떤 독자에게 맞는지 1문장)
  ```
- 스와이프 방향:
  - **위로** 또는 **옆으로** → 같은 분야의 다음 신간
- 좌상단 **🏠** 버튼 → 홈 화면(분야 선택)으로 복귀
- 상단 검색 바 항상 표시

---

### FR-003 카드 큐 관리

- 분야 진입 시 신간 5권 미리 로드 (Kakao Book API, 최신순 정렬)
- 마지막 카드 도달 전 다음 5권 자동 로드 (무한 스크롤)
- 이미 본 책은 같은 세션 내 재등장 안 함

---

### FR-004 AI 2문장 요약

- Claude Haiku로 생성:
  ```
  1문장: 책의 핵심 내용 또는 인상적인 한 장면
  2문장: 이런 독자에게 맞다 (어떤 상황/취향의 사람)
  ```
- Firestore `books/{isbn}/aiSummary`에 캐싱
  - 캐시 히트: p95 ≤ 300ms
  - 캐시 미스: p95 ≤ 3초 (생성 중 skeleton 표시)

---

### FR-005 통합검색

- 홈 화면 및 숏츠 뷰 모두에서 항상 상단 고정
- Kakao Book API, 300ms 디바운스 자동완성
- 검색 결과 터치 → 해당 책을 숏츠 뷰 첫 카드로 표시
- 이후 스와이프 시 같은 분야(카카오 API 반환 카테고리 기준) 책으로 이어짐

## 4. UI/UX 디자인

### 4.1 디자인 원칙

- **따뜻한 파스텔톤 일관성:** 전체 서비스가 따뜻하고 부드러운 파스텔 색감을 유지한다. 차갑거나 채도 높은 색은 사용하지 않는다.
- **정돈된 그리드:** 분야 카드는 균일한 크기의 둥근 사각형으로 규칙적으로 배열한다. 여백과 간격이 일정해 시각적으로 깔끔한 인상을 준다.
- **모바일 First:** iOS Safari / Android Chrome 기준. 한 손 조작 최적화.

### 4.2 화면 구성

```
[홈 — 분야 선택]
  상단: 검색 바 (항상 고정)
  중단: 분야 카드 그리드 (2열)
        ┌──────────┐  ┌──────────┐
        │  자기계발  │  │   소설    │   ← 둥근 사각형 카드
        └──────────┘  └──────────┘   ← 분야마다 다른 파스텔 색
        ┌──────────┐  ┌──────────┐
        │  에세이   │  │ 경제/경영  │
        └──────────┘  └──────────┘
        ...

[숏츠 뷰 — 해당 분야 책]
  상단: 🏠 홈 | 분야명 | 🔍 검색  (항상 고정)
  중단: 책 표지 (전체 너비, 상단 ~55%)
  하단: 제목·저자·2문장 요약 (하단 ~45%)
```

### 4.3 분야 카드 디자인

- **형태:** 모서리가 충분히 둥근 사각형 (`border-radius: 16px` 이상), 그림자 없이 배경색만으로 구분
- **크기:** 2열 그리드, 가로 전체 너비의 절반 — 높이는 가로의 약 0.65배 (가로형 직사각형)
- **색상:** 분야마다 고유한 따뜻한 파스텔 색상 배정 (아래 분야별 색상표 참고)
- **텍스트:** 분야명을 카드 중앙에 배치, 폰트는 중간 굵기, 색상은 해당 파스텔 색의 어두운 톤

#### 분야별 파스텔 색상표

| 분야 | 배경색 | Hex |
|---|---|---|
| 자기계발 | 복숭아 | `#FFDAC1` |
| 소설 | 라벤더 | `#C3B1E1` |
| 에세이 | 민트 | `#B5EAD7` |
| 경제/경영 | 레몬 | `#FFEAA7` |
| 역사 | 스카이 | `#B8E4F9` |
| 과학 | 라임 | `#C7E9B0` |
| 인문 | 로즈 | `#F7C5C5` |
| 사회 | 피치 | `#FAD4B4` |
| 예술 | 퍼플 | `#D4C5F9` |

### 4.4 전체 컬러 팔레트

| 역할 | 색상 | Hex |
|---|---|---|
| Background | 아이보리 (따뜻한 흰색) | `#FAFAF7` |
| Surface | 화이트 | `#FFFFFF` |
| Text Primary | 다크 그레이 | `#2D2D2D` |
| Text Secondary | 미디엄 그레이 | `#6B7280` |
| Border | 라이트 그레이 | `#E5E7EB` |
| Accent — 분야 없음 | 라벤더 | `#C3B1E1` |

> 분야 카드 외 UI 요소(버튼, 헤더, 카드 배경 등)는 위 팔레트를 기준으로 하되, 항상 따뜻하고 부드러운 인상을 유지한다.

### 4.5 인터랙션

| 제스처 / 액션 | 동작 |
|---|---|
| 분야 카드 터치 | 해당 분야 숏츠 뷰 진입 |
| 위·옆으로 스와이프 | 같은 분야 다음 책으로 전환 |
| 🏠 터치 | 홈 화면(분야 선택)으로 복귀 |
| 검색 바 터치 | 검색 입력, 결과 목록 표시 |

## 5. 기술 아키텍처

### 5.1 스택

| 항목 | 선택 |
|---|---|
| 프레임워크 | React 19 + TypeScript |
| 번들러 | Vite |
| 스타일링 | Tailwind CSS v4 |
| 라우팅 | React Router v7 |
| 상태 관리 | Zustand |
| 백엔드/DB | Firebase (Firestore — AI 요약 캐시만) |
| AI | Claude Haiku |
| 도서 API | Kakao Book API |

### 5.2 Firestore 데이터 모델

```
books/{isbn}
  └─ aiSummary: { line1, line2, cachedAt }
```

로그인·유저 데이터 없음. 캐시 전용으로만 사용.

### 5.3 AI 요약 생성 파이프라인

```
카드 로드 요청
    ↓
Kakao API → 책 메타 (제목·저자·소개·표지)
    ↓
Firestore 캐시 확인
  ├─ hit  → 캐시 반환
  └─ miss → Claude Haiku 호출 (2문장 생성)
             → Firestore 저장
    ↓
카드 렌더링
```

## 6. 비기능 요구사항

| 항목 | 기준 |
|---|---|
| AI 요약 (캐시 히트) | p95 ≤ 300ms |
| AI 요약 (캐시 미스) | p95 ≤ 3초 |
| 카드 전환 애니메이션 | 60fps |
| 로그인 불필요 | 전체 기능 비로그인 이용 가능 |
| 모바일 First | iOS Safari / Android Chrome |
